## Evening Session Summary
Extended the storage layer with correct-answer tracking, a smart session selection algorithm, shuffled answer choices, and a real progress screen.

## Implemented (Evening)

- **Correct Answer Persistence** (`utils/storage.ts`, `app/answer-feedback.tsx`):
  - Added `correctQuestions: string[]` to the `UserStats` interface.
  - Replaced `updateMissedQuestions` with `updateQuestionStats(questionId, isCorrect)`, which now atomically updates both `correctQuestions` and `missedQuestions` — adding to one and removing from the other on each answer.
  - Updated `loadStats` default/fallback value to include the new field so existing stored data won't break.

- **Smart Daily Session Algorithm** (`utils/storage.ts`):
  - Replaced the old "shuffle everything, pick 5" logic with a structured 5-question composition:
    - **1 Confidence Check** — a previously-correct question not seen recently, to confirm retention.
    - **1 Resurfaced Weak Spot** — scored by a weighted heuristic: direct misses score +2, any question in a repeatedly-missed *subdomain* scores additional points. This biases selection toward problem subdomains without hammering the exact same question every day.
    - **3 New Questions** — unseen questions not in missed or correct sets.
  - Each slot has a graceful fallback (e.g., if nothing qualifies as "new", fill from any unselected question), so the session always reaches 5 questions.
  - Final 5 are shuffled so the user cannot detect the quiz structure from question order.
  - Weak-subdomain scoring is computed on-the-fly from existing `missedQuestions` + `seedQuestions` data — no additional storage fields required.

- **Shuffled Answer Choices** (`app/quiz.tsx`):
  - Choices were previously rendered in their hardcoded seed order, making the correct answer predictable.
  - Now uses `useMemo` keyed on `question.id` to shuffle choices into a randomized `{ text, originalIndex }` array on each question load.
  - The `originalIndex` is passed to `answer-feedback.tsx` on selection, so evaluation against `question.correctIndex` remains correct regardless of display order.
  - `useMemo` is called unconditionally (before early returns) to satisfy React's Rules of Hooks.

- **Progress Screen** (`app/(tabs)/progress.tsx`):
  - Replaced placeholder text with a live metrics dashboard, refreshed on every tab focus via `useFocusEffect`.
  - **Top stats row**: Sessions completed, overall accuracy %, and total unique questions tracked.
  - **Domain Mastery section**: Dynamically computes per-domain correct/total counts from `correctQuestions` and `missedQuestions`, displaying each domain's accuracy with green (≥ 50%) or red (< 50%) color coding, sorted strongest-to-weakest.
  - **Weak Spots section**: Shows a plain-language count of identified weak-spot questions with a brief explanation of how Reverb resurfaces them.
  - Migrated to use `ScreenContainer` (scrollable), design tokens, and `CommonStyles` consistent with the rest of the app.

- `app/quiz.tsx` — shuffled choices via `useMemo`, moved early returns after hook calls
