# Reverb - 2026-05-15

## Session Summary

Today we added question-level feedback to the quiz flow so signed-in users can rate a question with thumbs up or thumbs down, optionally tag thumbs-down feedback with reason codes, and save that feedback to Supabase.

## Implemented Today

- **Question Feedback UI**:
  - Added a reusable quiz feedback control below the answer explanation.
  - Added thumbs-up and thumbs-down rating buttons using the existing theme system and Lucide icons.
  - Shows reason-code chips only after the user selects thumbs down.
  - Added user-facing labels for every allowed `reason_codes` value.
  - Disables feedback controls while a save is in flight to avoid overlapping writes.
  - Resets local feedback state when the quiz advances to a new question.

- **Question Feedback Persistence**:
  - Added a typed `data/question-feedback.ts` helper for writing question feedback to Supabase.
  - Uses the regenerated Supabase types for `public.question_feedback`.
  - Saves `user_id`, `question_id`, `rating`, and `reason_codes`.
  - Uses `upsert` with `onConflict: "user_id,question_id"` so each user can only submit one feedback row per question.
  - Lets Postgres own the feedback row ID instead of generating IDs in the client.

- **RLS and Constraint Verification**:
  - Confirmed the remote table has SELECT, INSERT, and UPDATE policies scoped to `auth.uid() = user_id`.
  - Confirmed the new unique constraint exists as `UNIQUE (user_id, question_id)`.
  - Confirmed the table has checks for `rating` and the allowed `reason_codes` array values.
  - Fixed the reason-chip save path by targeting the new unique constraint instead of attempting primary-key-based upserts.
