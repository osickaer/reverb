# Reverb - 2026-04-27

## Session Summary

Today we moved the Daily 5 from a local per-user generated session to a shared Supabase-backed daily pack, then started persisting completed daily results back to Supabase. We also refined the quiz interaction so answers are selected first and only locked in after an explicit submit step.

## Implemented Today

- **Daily Pack Runtime**:
  - Added app runtime support for fetching the current `daily_packs` row by `day_key`.
  - Loads ordered `daily_pack_questions` rows from Supabase and uses those question IDs for the Daily 5.
  - Stores the `dailyPackId`, `dailyPackTitle`, and ordered `dailyPackQuestionIds` on the local daily session.
  - Refreshes the daily pack before reusing an existing local session so the app does not keep stale question IDs when today's Supabase pack changes.
  - Rebuilds the local daily session when the stored pack or ordered questions no longer match Supabase.

- **Generated Supabase Types Integration**:
  - Updated the app to use the regenerated `types/database.types.ts` schema for `daily_packs`, `daily_pack_questions`, `daily_results`, and `daily_answers`.
  - Typed the daily-pack query helpers against the generated table row types.
  - Preserved compatibility with older local session data while adding the new pack metadata fields.

- **Daily 5 UI Updates**:
  - Removed the old `new`, `missed`, and `resurfaced` labels from Daily 5 quiz questions.
  - Updated the Home preview chips to show the category mix for the shared daily pack, such as `1 geography`, `2 math`, and `2 history`.
  - Kept older session history and summary screens tolerant of old saved sessions that still have question-type metadata.

- **Quiz Submit Step**:
  - Changed quiz answer behavior so tapping a choice only marks it as selected.
  - Added a `Submit Answer` button that appears after a selection.
  - Moved scoring, persistence, stats updates, feedback haptics, and feedback display behind the submit action.
  - Kept the submit button on the app's normal primary blue instead of matching the question category theme.

- **Daily Results Persistence**:
  - Added a Supabase result-saving helper for completed Daily 5 sessions.
  - Saves one `daily_results` row per signed-in user and daily pack.
  - Saves one `daily_answers` row per pack question, including selected index, correctness, position, and the referenced `daily_pack_question_id`.
  - Updates an existing result for the same user and daily pack if the session is completed again.
  - Keeps local completion behavior working even if the remote save fails, logging the Supabase error for follow-up.

## Files Changed

- `data/daily-packs.ts` - New daily-pack fetch/cache helper using `daily_packs` and `daily_pack_questions`.
- `data/daily-results.ts` - New daily result and answer persistence helper for `daily_results` and `daily_answers`.
- `types/database.types.ts` - Regenerated Supabase database types including the daily-pack and result tables.
- `utils/storage.ts` - Daily session initialization now uses Supabase packs, stores pack metadata, and saves completed Daily 5 results remotely.
- `app/(tabs)/index.tsx` - Home Daily 5 preview now summarizes the pack's category mix.
- `app/quiz.tsx` - Daily questions no longer show old question-type badges, and answer submission now uses a select-then-submit flow.
- `app/session-summary.tsx` - Summary remains compatible with older sessions while hiding the old question-type breakdown for new daily packs.
