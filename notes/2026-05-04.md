  - Removed the previous placeholder Leaderboard card and supporting copy.

  - Added a red badge to the friends button showing the number of incoming friend requests.
  - Refreshes the incoming request count when the Leaderboard tab gains focus.

- **Friend Management Sheet**:
  - Replaced the centered popup modal with a slide-up page-style modal.
  - Added a header with the friends icon and `Friends` title.
  - Kept `Add friend` in its own section card.
  - Moved `Friends`, `Incoming`, and `Outgoing` into a tabbed subsection outside of cards.
  - Added a red badge to the `Incoming` tab when pending incoming requests exist.

- **Friend Request Backend Wiring**:
  - Added typed Supabase helpers using the generated `friendships` table types.
  - Loads friends, incoming requests, and outgoing requests for the signed-in user.
  - Checks that a username exists before sending a friend request.
  - Prevents self-requests and duplicate pending or accepted friend relationships.
  - Inserts pending friend requests into the `friendships` table.
  - Shows sent requests immediately in the `Outgoing` tab.
  - Allows incoming requests to be accepted or rejected.
  - Allows accepted friends to be removed.

- **Safety and Feedback**:
  - Shows success and error feedback after friend actions.
  - Added loading states for sending, loading friends, and request actions.
  - Added a native confirmation dialog before removing a friend.
  - Keeps the Leaderboard button badge and modal Incoming tab badge in sync after loading, accepting, or rejecting requests.

- **Leaderboard Data Helper**:
  - Added a typed `data/leaderboard.ts` helper for the Leaderboard tab.
  - Loads the selected day from `daily_packs.day_key`.
  - Loads the signed-in user, accepted friends, matching `daily_results`, and ordered `daily_answers`.
  - Loads the selected pack's ordered `daily_pack_questions` and each question's domain so everyone is compared against the same question sequence.
  - Calculates each visible user's streak client-side from recent `daily_packs` and `daily_results`.
  - Returns a compact `LeaderboardSnapshot` with current-user and friend rows ready for the UI.

- **Leaderboard Tab Surface**:
  - Added the `Leaderboard` heading while keeping the existing friends action and incoming request badge.
  - Added a date switcher that defaults to the current local date and allows browsing backward/forward through past dates.
  - Added loading, empty, and error states for leaderboard data.
  - Added a current-user Daily 5 result card.
  - Added a `Friend Results` section that shows accepted friends, their score, streak, and whether they played the selected day.

- **Friend Result Row Design**:
  - Iterated from green/red dots to a cleaner single-line row.
  - Removed usernames from friend rows to reduce clutter.
  - Kept the row compact with avatar, display name, ordered score strip, and score.
  - Moved streak into a small overlay badge on the avatar.
  - Muted avatar styling to avoid too many competing colors.
  - Tuned the streak badge to use a compact solid orange fill with no thick white border and balanced padding.

- **Ordered Domain Icon Score Strip**:
  - Replaced plain dots/pills with ordered category-icon segments using `constants/domain-themes.ts`.
  - Preserves the actual Daily 5 question order so users can compare which question positions friends got right or wrong.
  - Shows correct answers as light green outlined segments with green icons.
  - Shows incorrect answers as light red outlined segments with red icons.
  - Shows unanswered positions as muted gray segments.
  - Uses the same domain icon sequence for every user on the selected day.
