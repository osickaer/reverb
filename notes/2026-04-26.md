# Reverb - 2026-04-26

## Session Summary

Today we finished the first backend-backed question-bank migration path and moved the app toward shared cloud content. Reverb now reads active questions from Supabase instead of the local TypeScript question arrays, and we added scripts for importing the original question data and generating shared daily packs.

## Implemented Today

- **Supabase Question Import Script**:
  - Added a Python script to convert the original flat TypeScript question data into the new `public.questions` table shape.
  - Maps camelCase app fields to database snake_case fields, including `conceptId` -> `concept_key`, `correctIndex` -> `correct_index`, and `learnMoreQueries` -> `learn_more_queries`.
  - Validates required fields, difficulty values, answer indexes, duplicate question IDs, and importance values before writing.
  - Supports dry runs by default and only writes to Supabase when `--execute` is passed.
  - Uses `SUPABASE_SERVICE_ROLE_KEY` for writes and reads the project URL from `SUPABASE_URL` or `EXPO_PUBLIC_SUPABASE_URL`.

- **Cloud Question Bank Runtime**:
  - Added checked-in Supabase database types for the `questions` table.
  - Typed the shared Supabase client with the database type.
  - Replaced the app's local merged `seedQuestions` usage with async Supabase reads from `public.questions`.
  - Added a small cache in `data/questions.ts` for active cloud questions.
  - Preserved the app-facing `Question` model by mapping Supabase rows back into the existing camelCase shape.

- **Quiz and Progress Screens Updated**:
  - Updated quiz startup, answer feedback, session summary, session history, home, progress, and storage flows to load questions asynchronously.
  - Replaced direct local question lookups with `fetchQuestions`, `getQuestionById`, or question maps built from the cloud result.
  - Fixed the pre-existing conditional `useDomainTheme` hook-order lint issues in session summary and session history while touching those screens.

- **Daily Pack Generator**:
  - Added `scripts/generate_daily_packs_to_supabase.py` for shared daily packs.
  - Generates 31 packs by default: today plus the next 30 days.
  - Each pack has 5 questions with the shape `easy`, `medium`, `medium`, `hard`, `hard`.
  - Uses deterministic UUIDs based on date and position so reruns are stable.
  - Upserts `daily_packs` by `day_key`.
  - Deletes and regenerates join rows for generated packs before inserting `daily_pack_questions`, so algorithm changes can be reapplied cleanly.
  - Balances domains as much as the available difficulty pool allows.
  - Added `--min-repeat-gap-days`, defaulting to 7, so the same question is not reused too close together when alternatives exist.

## Commands

- Import questions dry run:
  - `python scripts/import_questions_to_supabase.py`
- Import questions to Supabase:
  - `python scripts/import_questions_to_supabase.py --execute`
- Generate daily packs dry run:
  - `python scripts/generate_daily_packs_to_supabase.py`
- Generate and upsert daily packs:
  - `python scripts/generate_daily_packs_to_supabase.py --execute`
- Tune repeat spacing:
  - `python scripts/generate_daily_packs_to_supabase.py --min-repeat-gap-days 10`
