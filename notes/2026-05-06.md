# Reverb - 2026-05-06

## Session Summary

Today we fixed an account-switching bug that made Daily 5 completion state leak between test accounts, added pull-to-refresh across the main tabs, and continued refining the Leaderboard tab into a simpler consolidated Daily 5 results surface with a new weekly performance rollup.

## Implemented Today

- **Account-Scoped Local Storage**:
  - Fixed Daily 5 and stats state leaking across accounts after logout/login.
  - Scoped local AsyncStorage keys by the active Supabase user ID.
  - Added an explicit auth-context hook to keep storage scoped to the current signed-in user.
  - Left old unscoped storage data unmigrated to avoid accidentally importing one account's local session state into another account.
  - Updated storage debug helpers so clearing Reverb storage removes both legacy global keys and new user-scoped keys.

- **Pull-To-Refresh Support**:
  - Added shared `RefreshControl` support to `ScreenContainer`.
  - Added pull-to-refresh on Home.
  - Added pull-to-refresh on Progress.
  - Added pull-to-refresh on Leaderboard.
  - Added pull-to-refresh on Profile.

- **Leaderboard Structure Cleanup**:
  - Removed the streak badge that overlaid avatars.
  - Consolidated the current user result and friend results into one `Daily 5 Results` list.
  - Pinned the current user's row at the top of the list.
  - Removed the separate current-user result card and separate `Friend Results` section.
  - Kept the date selector as a card-style widget after trying a flatter page-level version.

- **Current User Row Treatment**:
  - Replaced the `You` display-name override with the user's real display name.
  - Moved the `You` label into a subtle subtitle under the display name to save horizontal space.
  - Gave the current user's row a neutral `surfaceAlt` background so it reads as pinned without using awkward blue accents.
  - Made the current user's display name bolder than friend names.

- **Leaderboard Row Density**:
  - Removed the redundant `x/5 correct` line under display names.
  - Reduced avatar size slightly.
  - Reduced score segment size and spacing slightly.
  - Restored row padding and gaps after a tighter pass made the layout feel cramped.
  - Kept the row layout compact enough to handle longer display names better while preserving natural spacing.

- **Weekly Leaderboard Section**:
  - Added a new `This Week` section below `Daily 5 Results`.
  - Shows the current Monday-Sunday week range, such as `May 4-10`.
  - Aggregates only this week's completed Daily Pack results for each user and friend.
  - Displays weekly score totals as `score/total` plus a weekly percentage.
  - Adds streak pills back in this weekly context using the same orange flame styling as Home and Progress.
  - Pins the current user at the top with the same subtle `surfaceAlt` row background and `You` subtitle.
  - Sorts friends by weekly percentage, then score, then total answered, then display name.
  - Aligned leaderboard streak behavior with Home and Progress so a user's streak carries through yesterday until today's Daily 5 is completed.

- **Friend Modal Stability**:
  - Fixed the friend management modal twitching between loaded content and `Loading friends`.
  - Root cause was a render loop: the modal reloaded friendships whenever the parent passed a new `onIncomingRequestCountChange` callback, and that callback refreshed the leaderboard.
  - Stabilized `loadFriendships` and stored the latest incoming-request callback in a ref so count updates still flow without retriggering the modal's initial load effect.

- **Weekly Leaderboard Copy**:
  - Clarified the `This Week` subtitle so it explains the row metrics without adding extra column labels.
  - Current wording is `May 4-10 · lifetime streak and weekly score`, pairing the date range with the meaning of the streak and weekly score values.
